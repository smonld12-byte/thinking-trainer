# 信息处理训练模式 — 设计文档

## 概述

在现有 7 种思维训练模式基础上，新增第 8 种模式「信息处理」，训练**输入→提炼→重组→输出**的信息处理能力。源于抖音视频「语言组织能力和复述能力怎么提高」的 14 天阶梯训练法。

现有模式练「给定观点→结构化输出」（输出端），新模式练「接收信息→提炼重组→输出」（输入端），两者互补。

## 目标受众

所有人（不限于设计师），AI 生成素材覆盖多领域。

## 四个子阶段

每个子阶段是一个独立的训练入口，用户可从任意阶段开始，也可按 1→2→3→4 递进：

| 阶段 | 名称 | AI 生成素材 | 用户输入 | AI 评分维度 | 阅读限时 |
|------|------|-----------|---------|------------|---------|
| 1 | 关键词提取 | ~300字短文 | 3-5个关键词 | 核心信息命中率、简洁度 | 无 |
| 2 | 一句话总结 | ~300-500字短文 | 一句话概括（≤50字） | 逻辑完整度、关键信息覆盖、是否冗余 | 无 |
| 3 | 三点复述 | 模拟对话/观点陈述（~500字） | 三点结构化复述 | 结构清晰、信息还原度、是否曲解 | 无 |
| 4 | 计时复述+评论 | 模拟采访/辩论对话（~600字） | 限时复述+个人评论 | 信息准确度、评论深度、时间效率 | 阅读3分钟 |

## 双赛道策略

### AI 驱动赛道
- AI 实时生成所有素材（短文、对话脚本）
- AI 生成参考答案（标准关键词/总结/复述要点）
- AI 评分 + 反馈（对标现有 `triggerAiScore()` 的 JSON 返回格式）
- 领域随机，也可通过 `seed` 参数保证不重复

### 自主练习赛道
- 预置 12-16 篇短文/对话脚本（每阶段 3-4 篇）
- 提供参考结构和提示（类似现有的 `structure` 字段）
- 手动自评，不依赖 AI

## AI 交互流程

### 阶段 1（关键词提取）示例
```
System: 你是一位沟通训练教练。请生成一篇约300字的短文（领域随机），
同时生成3-5个标准关键词作为评分参考。

用户看到: 短文内容（~300字）
用户输入: 3-5个关键词
AI评分: { keywords: [...], accuracy: 0-10, feedback: "...", missedCore: "..." }
```

### 阶段 2（一句话总结）示例
```
用户看到: 短文内容（~300-500字）
用户输入: 一句话总结
AI评分: { summaryQuality: 0-10, coreIdea: "...", feedback: "...", isComplete: bool }
```

### 阶段 3（三点复述）示例
```
用户看到: AI生成的对话脚本/观点陈述（~500字）
用户输入: 三个要点（结构化）
AI评分: { structureScore: 0-10, accuracyScore: 0-10, feedback: "...", referencePoints: [...] }
```

### 阶段 4（计时复述+评论）示例
```
流程:
1. 显示素材 → 启动3分钟阅读倒计时
2. 倒计时结束 → 自动切换到输入界面，开始3分钟输出倒计时
3. 用户输入复述+评论
4. 提交 → AI评分

AI评分: { accuracy: 0-10, commentDepth: 0-10, timeEfficiency: 0-10, feedback: "..." }
```

## UI 设计

### 首页卡片
- 图标: 📡
- 模式名称: 信息处理
- 副标题: "提取关键词 → 总结 → 复述 → 评论，四阶递进训练"
- 布局: 第 8 张卡片，grid 从 2×4 变为两行 4 列或保持 2 列自适应

### 训练界面
- 遵循现有 `.topic-card` + `.input-group` 结构
- 阶段 1-3：素材只读展示区 + 输入框 + 提交按钮
- 阶段 4：素材区 + 倒计时显示 + 输入区（阅读结束自动切换）
- 评分展示：复用现有 `.review-header` + `.compare-section` 结构

### 阶段选择
- 训练界面顶部显示 4 个子阶段 tab（类似阅览室的分类 tab）
- 默认选中阶段 1

### 音频标注
- 阶段 3/4 素材区顶部：`🔊 模拟听读材料（Beta）——建议让朋友念给你听，后续将支持语音播放`

## 兼容性约束

- **不引入新依赖**：单文件 HTML，纯原生 JS
- **不破坏现有结构**：exerciseTypes 新增 `infoProcess` 条目，现有 7 种模式不动
- **API 复用**：使用现有的 `getApiKey()` / `getApiUrl()` / `getModel()` 和 Anthropic Messages API 调用模式
- **设计风格一致**：沿用现有 CSS 变量和组件样式

## 实施范围

### Phase 1（本次实施）
- 新增 infoProcess 模式：4 个子阶段
- AI 赛道：素材生成 + 评分反馈（4 种评分 prompt）
- 自主练习：每阶段 3 篇预置（共 12 篇）
- 阶段 4 计时器：阅读倒计时 + 输出倒计时
- 首页卡片布局适配（8 个模式）

### Phase 2（后续）
- TTS 语音播放（Web Speech API）
- 更多预置素材（用户贡献或 AI 批量生成）
- 14 天打卡计划 UI

## 设计决策记录

| 决策 | 选型 | 原因 |
|------|------|------|
| 音视频素材 | 文字稿（beta） | 第一版轻量，后续加 TTS |
| 受众 | 所有人 | 用户明确要求，领域随机 + 可扩展 |
| AI 深度 | 全流程（素材+答案+评分） | 对标现有 AI 赛道体验 |
| 阶段4计时 | 阅读限时 + 输出限时 | 用户选择，制造适度压力 |
| 模式名称 | 信息处理 | 中性准确，概括四个子阶段的共性 |
